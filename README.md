# JS-7-Tips
7 Tips for Becoming a Competent JavaScript Developer.

1. Understand Plain JavaScript: 
Understanding core principles like scopes, closures, syntax, etc., is crucial but easily 
discounted by developers who focus on learning popular tools and frameworks like React, 
Vue, or Angular. Someone who can see a fragment of JavaScript code and understand what is 
going on will be able to learn new frameworks relatively quickly, but the opposite is not 
necessarily true.

2. Read Quality Content
It is easy to get lost in "Tutorial Hell," so focusing on finding quality content is key. 
Books are a great resource because the author has to put a lot of work into collecting 
and organizing the information. Unlike many free online resources, you cannot write a 
book overnight. Another benefit to reading books over watching tutorial videos is the 
focus required to sit down and read a book. It is easy to do other things while watching 
videos, but choosing a book and sitting down to read it allows your full attention to be 
devoted to learning content. I have found that to be true in class as well. Reading the 
textbooks has been instrumental in learning and building upon knowledge each week.

3. Familiarize Yourself With Different Frameworks
I have only ever used React because I read that it was the most popular, and few of my 
courses focused on it, so this section stuck out to me. I agree that it is necessary to 
be able to formulate your own opinions on the pros and cons of different frameworks. Van 
der Put shared his experience of interviewing potential developers and asking them why 
they preferred working with specific frameworks over the others. He said, "Nine times 
out of ten, I'm met with silence. They haven't even tried." He goes on to urge: "Try. 
Experiment. Play. Form some educated opinions. Your opinions. And bring them with you 
on your journey to becoming a competent JavaScript developer."

4. Understand HTML and CSS
Sometimes I feel like I spend too much time focusing on HTML and CSS when there is so 
much to learn, but this made me feel better about that choice. HTML and CSS are the 
building blocks of the internet. Van der Put explains, "React, Vue, Angular? As awesome 
as they can be, they all "spit out" HTML in the end." While it is not necessarily critical 
as a backend developer, producing websites requires a concrete understanding of HTML, CSS, 
and JavaScript and how they work together. 

5. Respect Senior Developers
Working with mentors and peers is such an essential source of learning. It is one of the 
main reasons I decided to take classes instead of teaching myself with books and videos. 
Senior developers are people who have been doing what I am trying to learn for years, 
and any opportunities to learn from them are beneficial. I am a huge believer in the 
importance of learning through others' shared experiences, and becoming a competent 
JavaScript developer is no exception.

6. Be Curious, Investigate, and Gain Knowledge
I consider myself a problem-solver, so I feel very aligned with this core personality 
trait description. I am curious by nature; however, I am also a little impatient. I have 
to be better about actively reminding myself that my main goal should not be fixing bugs 
or completing tasks but gaining more knowledge every day.

7. Write Code and Write Some More
Like anything, becoming an expert means practicing. In this case, writing code is the 
only way to become a better coder. A tip I will take away from the article by van der 
Put is not copying and pasting code even when the tutorials provide it. Seeing code and 
how it works is important, but writing it yourself is more so.
